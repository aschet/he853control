HE853 Control Project

Introduction

This project is dedicated to create a set of Windows tools for the HE853 USB dongle of the Home Easy home automation products since the vendor does not provide an API. Most information this project is based on was gathered through reverse engineering. For more resources please refer to the project page: https://sourceforge.net/projects/he853control/

Provided Tools (.NET Framework 2.0 required):

  • HE853.dll, LGPL licensed library
  • HE853.Util.exe, GPL licensed command line utility
  • HE853.App.exe, GPL licensed simple GUI application
  • HE853.Service.exe, GPL licensed service

Download

The current package can be obtained from the project download page.

Usage Library

The library is written in C# and provided as .NET assembly (HE853.dll) which works currently only with x86 platform configurations.

C# Sample

Reference HE853.dll in your project and use the following code. 

HE853.Device device = new HE853.Device();
if (device.Open())
{
    device.On(1001);
    device.Off(1001);
    device.Close();
}

C++ COM Interop Sample

Register HE853.dll .NET with the Assembly Registration-Tool (Regasm.exe). Import the resulting type library with the #import directive and use the following code. 

HE853::IDevicePtr device(__uuidof(HE853::Device));
if (device->Open())
{
  device->On(1001);
  device->Off(1001);
  device->Close();
}

Usage Command Line Utility

Home Easy HE853 Control Utility by Thomas Ascher

Usage: HE853.Util <command> <device_code>

<command> := ON | OFF | 1..100
<device_code> := 1..6000

The device code has to programmed to a receiver first.
To program the code hold the learn button on the receiver for
about 2 seconds and send an ON command with the specific code.

Example: HE853.Util ON 1001

Usage Service

The service application HE853.Service is for advanced users. It is meant when multiple applications have to access the HE853 dongle in a synchronized manner. Both the utility and the GUI application support the /service switch that makes them use the service instead of using the device directly. To install the service you have to use Installer Tool (Installutil.exe). The service is configured to not start automatically per default.
